# SEO Phase 1 verification

## Existing foundation reused

The app already had a single client-side `Seo` head manager, route metadata, canonical tags, Open Graph and Twitter tags, JSON-LD, a generated sitemap, `robots.txt`, a branded 404/product-not-found state, favicon, and Netlify SPA fallback. Phase 1 refines those paths rather than adding duplicate tags or schema scripts.

## Public metadata

- `/`: Koch Rajbanshi heritage homepage metadata; Organization and WebSite JSON-LD.
- `/shop`: shop-specific title, description, and canonical.
- `/about`: Our Story metadata.
- `/contact`: enquiries and support metadata.
- `/policies`: public policies metadata.
- `/product/:slug`: product name, sanitized description, primary HTTPS image, price metadata, Product/Offer and Breadcrumb JSON-LD. Genuine approved reviews are included only when returned by the review service.

The default social image is `/images/social/home-share-v1.jpg` (1200 × 630). Canonicals and social images resolve against `https://houseofpatani.com`.

## Noindex routes

Cart, checkout, login, OTP verification, account routes, order lookup, order confirmation, admin routes, invalid products, and unknown routes emit `noindex`. These routes are excluded from the sitemap.

## Files and generation

- Sitemap: `/public/sitemap.xml`, regenerated by `npm run generate:sitemap` during every build.
- Robots: `/public/robots.txt`, generated alongside the sitemap.
- Product sitemap entries currently use the safe local product seed source. If Supabase becomes the only catalog source, replace this with a build-time read using a restricted public catalog credential; never expose a service-role key.

## Netlify prerendering

Client-side metadata is not sufficient for every social crawler. In the Netlify dashboard, enable Netlify Prerendering for the production site. Do not add a custom SPA SSR rewrite. Test `/`, `/shop`, `/about`, and at least two active `/product/:slug` URLs. Keep admin, account, checkout, and order routes out of prerender testing and indexing.

To verify, request a deployed product URL with a crawler user agent or use Netlify's prerender inspection. The raw response must contain that product's title, canonical, `og:image`, and Product JSON-LD—not only the homepage defaults.

## Manual checks after deployment

1. View rendered `<head>` while navigating between two products; confirm one title, description, canonical, robots tag, and JSON-LD set.
2. Test JSON-LD with Schema.org Validator or Google Rich Results Test.
3. Confirm invalid product and unknown routes emit `noindex` and no Product schema.
4. Open `/sitemap.xml` and `/robots.txt`; confirm production HTTPS URLs only.
5. Use Facebook Sharing Debugger and WhatsApp link sharing after prerendering is enabled. Refresh Facebook's scrape after metadata changes.

## Product social preview images

Product pages derive a 1200x630 JPEG social-preview URL on demand from the existing primary Supabase product image through Netlify Image CDN. No per-product social asset is generated or stored, and gallery and JSON-LD images continue to use the originals. Netlify caches each unique transformation at the edge, so Image CDN transformation and delivery usage should be monitored against the site's current Netlify plan.

After deploying a metadata or source-image change, Meta and WhatsApp may continue to show a cached preview. Run **Scrape Again** once in Meta Sharing Debugger when necessary. If a primary image is missing or is outside the allowlisted product-image bucket, metadata falls back to the original valid image or the branded default rather than emitting an empty image URL.
6. Do not submit the sitemap to Search Console and do not add analytics during Phase 1.

## Social preview cache checks

After deploying metadata changes, test a product URL that has not been shared before. Meta and WhatsApp can retain an older preview: use Meta Sharing Debugger and choose **Scrape Again**, then resend the URL in a new WhatsApp chat. A temporary harmless query parameter may be used only to test cache behavior; canonical URLs must remain clean and unchanged.
